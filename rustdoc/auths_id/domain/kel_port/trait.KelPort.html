Expand description

Domain port for Key Event Log operations.

Abstracts how KERI events are stored and retrieved. The domain provides and receives typed Event values without knowledge of the backing store (Git refs, database, in-memory map).

Args:

  • prefix: The KERI prefix identifying the identity.

Usage:

ⓘ
use auths_id::domain::kel_port::KelPort;
use auths_id::keri::Prefix;

fn check_identity(kel: &dyn KelPort, prefix: &Prefix) -> bool {
    kel.exists(prefix)
}
