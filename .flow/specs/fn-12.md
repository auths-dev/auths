# Dependency Architecture Refactor

## Overview

Restructure workspace crate dependencies, distribute test utilities from the monolithic `auths-test-utils` into per-crate `test-utils` features, separate domain traits from infrastructure implementations in `auths-id`, and enable automated `cargo publish --workspace`.

## Scope

- **5 phases**: distribute test utils, clean up auths-id, consolidate auths-core, automate publishing, verification
- **All workspace crates affected**: 14 crates + deletion of `auths-test-utils`
- **Breaking changes**: Yes (pre-launch, acceptable)

## Approach

### Phase 1: Distribute test utilities into per-crate `test-utils` features

Each fake/mock/fixture moves to the crate that **defines the trait it implements**:

| Fake/Helper | Trait Source Crate | Target |
|---|---|---|
| `create_test_keypair`, `get_shared_keypair`, `gen_keypair` | `auths-crypto` (CryptoProvider) | `auths-crypto/src/testing.rs` |
| `DeterministicUuidProvider` | `auths-core` (UuidProvider) | `auths-core/src/testing/` |
| `InMemoryStorage` (6 port traits) | `auths-core` (BlobReader/Writer etc.) | `auths-core/src/testing/` |
| `MemoryEventSink` | `auths-telemetry` (EventSink) | `auths-telemetry/src/testing.rs` |
| `MockClock` | `auths-verifier` (ClockProvider) | `auths-verifier/src/testing.rs` |
| `FakeRegistryBackend`, `FakeAttestationSink/Source`, `FakeIdentityStorage` | `auths-id` (RegistryBackend etc.) | `auths-id/src/testing/` |
| Contract test macros (`registry_backend_contract_tests!`, `event_sink_contract_tests!`) | `auths-id`, `auths-telemetry` | Same crate as trait |
| Fixtures (`test_inception_event`, `test_attestation`) | `auths-id` types | `auths-id/src/testing/fixtures/` |
| Mocks (`MockIdentityStorage`, `MockAttestationSource`) | `auths-id` | `auths-id/src/testing/mocks/` |
| `FakeGitLogProvider`, `FakeGitDiagnosticProvider`, `FakeCryptoDiagnosticProvider` | `auths-sdk` (GitLogProvider etc.) | `auths-sdk/src/testing/` |
| `git_log_provider_contract_tests!` | `auths-sdk` (GitLogProvider) | `auths-sdk/src/testing/contracts/` |
| `init_test_repo`, `get_cloned_test_repo`, `copy_directory` | N/A (git2 helpers) | `auths-infra-git/src/testing.rs` |

**Pattern to follow**: `auths-core/src/lib.rs:66-67` — `#[cfg(any(test, feature = "test-utils"))] pub mod testing;`

### Phase 2: Clean up `auths-id` — remove infrastructure dependencies

**Critical finding**: Trait definitions (`AttestationSource`, `AttestationSink`, `IdentityStorage`, `ManagedIdentity`) are currently behind `#[cfg(feature = "git-storage")]` alongside their git implementations. Phase 2 must:
1. First **ungate trait definitions** (split trait def from git impl in each module)
2. Then move git implementations to `auths-storage`
3. Then remove `git-storage` feature from `auths-id`

### Phase 3: Consolidate `auths-core` role — DECISION: Accept status quo (Option 3)

**Decision**: Accept `auths-core` depending on `auths-verifier`. The dependency is clean (no reverse dep — auths-verifier only depends on auths-crypto), and the 20 files in auths-core using types like `Prefix`, `Said`, `DeviceDID`, `Attestation`, `ClockProvider` represent genuine domain coupling. Extracting a shared types crate (Option 2) or moving types to auths-crypto (Option 4) are both too invasive for v0.1.0 with no practical benefit.

**Publish order**: `auths-crypto` → `auths-verifier` → `auths-core` → `auths-id` → `auths-storage` → rest.

**Architecture (actual layers)**:
```
Layer 0: auths-crypto
Layer 1: auths-verifier (depends on auths-crypto)
Layer 2: auths-core (depends on auths-crypto + auths-verifier)
Layer 3: auths-id, auths-policy (depend on auths-core)
Layer 4: auths-storage, auths-sdk (depend on auths-id)
Layer 5: auths-infra-git, auths-infra-http (depend on auths-sdk)
Layer 6: auths-cli (depends on everything)
```

**Decision on `auths-infra-git` → `auths-sdk`**: Accept. The coupling is minimal — a single file (`audit.rs`) imports 4 types (`GitLogProvider`, `CommitRecord`, `GitProviderError`, `SignatureStatus`). Moving these to `auths-core::ports::git` would be clean but is deferred post-v0.1.0.

### Phase 4: Automate publishing

`cargo publish --workspace` (Rust 1.90+, workspace uses 1.93). Consider `release-plz` for CI.

### Phase 5: Cleanup and verification

Remove any temporary workarounds, run full verification suite.

## Key Gaps Identified

1. **`auths-sdk` needs `test-utils` feature** — plan omits this. `FakeGitLogProvider`, `FakeGitDiagnosticProvider`, `FakeCryptoDiagnosticProvider` implement traits from `auths-sdk` and cannot go to `auths-id`.
2. **`auths-infra-git` depends on `auths-sdk`** (Layer 3) but plan puts it at Layer 2. `GitLogProvider` trait may need to move down.
3. **`session_store_contract_tests!`** references `auths_auth_server` (not a workspace member) — dead code, delete.
4. **Contract test macros need `$crate::` path rewriting** after moving to new crates.
5. **`FakeRegistryBackend.metadata()` calls `Utc::now()`** — violates clock injection rule, fix during migration.
6. **`auths-storage` has zero tests** — contract test infrastructure must be bootstrapped.
7. **CI needs `--all-features` testing** — `test-utils` modules not compiled by default `cargo nextest run --workspace`.
