# Epic: Refactor CLI Signing Orchestration into SDK

## Overview

Extract the monolithic signing logic from `crates/auths-cli/src/bin/sign.rs` (628 lines) into a composable `SigningWorkflow` in the SDK. Abstract subprocess and agent dependencies behind typed port traits. Eliminate `unwrap()`/`expect()` from library crate production code. This ensures future interfaces (FFI, desktop, WASM) can reuse the same signing mechanics without duplication.

## Scope

**In scope:**
- Define agent signing port trait in `auths-sdk/src/ports/`
- Add agent signing port to `AuthsContext` as optional field
- Create `SigningWorkflow` in `auths-sdk/src/workflows/signing.rs` with three-tier fallback
- Refactor `bin/sign.rs` to delegate to `SigningWorkflow`
- Abstract `Command::new("ssh-add")` in `auths-core/src/api/runtime.rs` behind a port trait
- Eliminate `unwrap()`/`expect()` in `auths-core` and `auths-sdk` production code
- Add `#![deny(clippy::unwrap_used, clippy::expect_used)]` lint enforcement

**Out of scope:**
- `run_verify()` and `run_delegate_to_ssh_keygen()` in `bin/sign.rs` (remain CLI-side)
- Pubkey cache migration (stays in CLI; passed as parameter to workflow)
- `AUTHS_PASSPHRASE` env var handling (stays in `PassphraseProvider` implementation)
- Passphrase caching TTL (stays in CLI; uses `ctx.clock.now()` at boundary)

## Approach

### Architecture decisions

1. **Port-first ordering**: Define port traits (Task 1) before building the workflow (Task 2). This avoids refactor-the-refactor.

2. **`AuthsContext` extension, not generics**: The workflow uses `&AuthsContext` (matching `sign_artifact_attestation` pattern at `signing.rs:327`), not the generics pattern from `DiagnosticsWorkflow`. Agent signing port is optional on `AuthsContext` with a `NoopAgentProvider` default for Windows/WASM.

3. **Preserve current Tier 2 behavior**: After auto-start + key load, fall through to Tier 3 (direct sign) ‚Äî NOT retry agent signing. This matches existing behavior and avoids a behavior change.

4. **Workflow returns data, CLI handles I/O**: `SigningWorkflow` returns `String` (PEM signature). File writing (`.sig`) stays in `bin/sign.rs`. Follows existing `DiagnosticsWorkflow` pattern.

5. **Distinguish `Unavailable` vs `Fatal` errors**: Agent not running ‚Üí fall through. Passphrase wrong ‚Üí halt chain. `process::exit(1)` replaced with `SigningError::PassphraseExhausted`.

6. **Safe-by-invariant `expect()` allowed**: `Lazy::new()` runtime creation and length-checked `try_into()` are acceptable. Focus audit on genuinely dangerous sites (mutex unwrap, user-facing errors).

### Key reuse points

- `AuthsContext` builder (`crates/auths-sdk/src/context.rs:92-114`) ‚Äî extend, don't duplicate
- `SigningError` enum (`crates/auths-sdk/src/signing.rs:22-39`) ‚Äî add new variants
- `sign_with_seed()` (`crates/auths-sdk/src/signing.rs:112-118`) ‚Äî compose into workflow
- `validate_freeze_state()` (`crates/auths-sdk/src/signing.rs:67-84`) ‚Äî call from workflow
- `PassphraseProvider` trait (`crates/auths-core/src/signing.rs`) ‚Äî inject via `AuthsContext`
- Agent client functions (`crates/auths-core/src/agent/client.rs`) ‚Äî wrap in CLI adapter
- Fake adapter pattern (`crates/auths-sdk/src/testing/fakes/`) ‚Äî follow for new fakes
- Test helper pattern (`crates/auths-sdk/tests/cases/helpers.rs`) ‚Äî extend for workflow tests
- CLI adapter pattern (`crates/auths-cli/src/adapters/system_diagnostic.rs`) ‚Äî follow for agent adapter

### Risks

- **Agent client is Unix-only** (`#[cfg(unix)]`): The `AgentSigningPort` trait must be platform-independent. CLI adapter wraps Unix-specific code; `NoopAgentProvider` used on other platforms.
- **`PassphraseProvider` trait changes**: Removing `Mutex::lock().unwrap()` from `CachedPassphraseProvider`/`UnifiedPassphraseProvider` may require `PassphraseProvider::on_incorrect_passphrase()` to return `Result` instead of `()`.
- **Witness server `expect()` calls**: Changing `create_receipt()` to return `Result` is a breaking API change. May need to be deferred or scoped carefully.
- **`bin/sign.rs` still needed**: After refactor, `bin/sign.rs` still handles arg parsing, verification, delegation, and file I/O. It becomes thinner but doesn't disappear.
