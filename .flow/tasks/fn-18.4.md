## Add fuzz_cesr_import target to auths-keri and CI nightly fuzz workflow

### New fuzz target

The only genuinely new fuzz target needed. Existing targets already cover attestation parse, DID parse, verify chain (auths-verifier), and policy eval (auths-policy).

Create `crates/auths-keri/fuzz/`:

```
crates/auths-keri/fuzz/
  Cargo.toml
  fuzz_targets/
    fuzz_cesr_import.rs
```

**Important:** This is NOT a symmetric roundtrip test. Raw bytes don't round-trip through CESR. The target fuzzes `import_cesr_to_events` with random bytes to ensure it never panics on malformed input.

```rust
#![no_main]
use libfuzzer_sys::fuzz_target;
use auths_keri::{CesrV1Codec, import_cesr_to_events};

fuzz_target!(|data: &[u8]| {
    let codec = CesrV1Codec;
    let _ = import_cesr_to_events(&codec, data);
});
```

## Add fuzz_cesr_roundtrip target to auths-keri and CI nightly fuzz step

### New fuzz target

Existing fuzz targets already cover:
- `auths-verifier/fuzz/`: attestation_parse, did_parse, verify_chain
- `auths-policy/fuzz/`: policy_eval

The only genuinely new target needed is CESR round-trip for auths-keri.

Create `crates/auths-keri/fuzz/`:

```
crates/auths-keri/fuzz/
  Cargo.toml
  fuzz_targets/
    fuzz_cesr_roundtrip.rs
```

The target: random bytes → attempt CESR decode → if decode succeeds, re-encode → compare. Should never panic.

```rust
#![no_main]
use libfuzzer_sys::fuzz_target;

fuzz_target!(|data: &[u8]| {
    if let Ok(decoded) = auths_keri::cesr::decode(data) {
        let re_encoded = auths_keri::cesr::encode(&decoded);
        assert_eq!(data, &re_encoded[..]);
    }
});
```

Adapt the exact function names to match auths-keri's public API for CESR encode/decode.

### Fuzz Cargo.toml

Follow the same pattern as `crates/auths-verifier/fuzz/Cargo.toml`:
- `[workspace] members = ["."]` (separate from main workspace)
- Depend on `auths-keri` via path
- Use `cargo-fuzz` and `libfuzzer-sys`

### CI nightly fuzz step

Add to `.github/workflows/ci.yml` (or a separate `fuzz.yml` nightly workflow):

```yaml

    - cron: '0 3 * * *'  # nightly at 3am UTC

    steps:

      - run: cargo install cargo-fuzz
      - run: cd crates/${{ matrix.target.crate }} && cargo +nightly fuzz run ${{ matrix.target.target }} -- -max_total_time=60
```
