## Add oidc-policy feature flag and auths-policy dependency

Add `auths-policy` as an optional dependency of `auths-oidc-bridge`, gated behind an `oidc-policy` feature flag. Follow the existing `github-oidc = ["dep:reqwest"]` pattern.

### Steps

1. In `crates/auths-oidc-bridge/Cargo.toml`:
   - Add `auths-policy = { workspace = true, optional = true }` under `[dependencies]`
   - Add `oidc-policy = ["dep:auths-policy"]` under `[features]`

2. In `crates/auths-oidc-bridge/src/lib.rs`:
   - Add `#[cfg(feature = "oidc-policy")] pub mod policy_adapter;` (module created in fn-16.2)

3. Add `BridgeError::PolicyDenied` variant in `crates/auths-oidc-bridge/src/error.rs`:
   - Feature-gate with `#[cfg(feature = "oidc-policy")]`
   - Map to HTTP 403 in `IntoResponse` impl
   - Include denial reason as structured field: `{ reason: String }`

4. Add policy config fields to `BridgeConfig` in `crates/auths-oidc-bridge/src/config.rs`:
   - `#[cfg(feature = "oidc-policy")] pub workload_policy_path: Option<PathBuf>` — path to a JSON policy file
   - `#[cfg(feature = "oidc-policy")] pub workload_policy_json: Option<String>` — inline JSON policy (alternative to file)
   - Add builder methods `with_workload_policy_path()` and `with_workload_policy_json()`
   - Do NOT store `CompiledPolicy` in config — it's a runtime artifact. The config holds the source; `BridgeState` compiles and holds the `CompiledPolicy`.

5. In `BridgeState::new()` (`crates/auths-oidc-bridge/src/state.rs`):
   - If policy path or JSON is configured, load/parse the JSON, compile via `auths_policy::compile_from_json()`, store as `Option<CompiledPolicy>` in state
   - Invalid policy JSON or compilation failure prevents bridge startup (fail-fast)
   - Log at `info` level on successful compilation, `warn` if no policy configured

### Files to modify
- `crates/auths-oidc-bridge/Cargo.toml`
- `crates/auths-oidc-bridge/src/lib.rs`
- `crates/auths-oidc-bridge/src/error.rs`

- `crates/auths-oidc-bridge/src/state.rs`
