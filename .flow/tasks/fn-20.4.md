## Add protocol state machine and update publish ordering

### Protocol state machine

Add a formal state machine to `auths-pairing-protocol` that makes the pairing lifecycle explicit for mobile consumers.

Current `PairingSession` uses `Option<EphemeralSecret>` for one-shot exchange. The state machine wraps this into typed states:

```rust
/// Transport-agnostic pairing protocol state machine.
///
/// Args: None (constructed via `initiate()`)
///
/// Usage:
/// ```ignore
/// let (protocol, token_bytes) = PairingProtocol::initiate(seed, device_pk)?;
/// // Caller sends token_bytes over their transport
/// // Caller receives response_bytes from peer
/// let result = protocol.complete(response_bytes)?;
/// ```
pub struct PairingProtocol {
    session: PairingSession,
    state: ProtocolState,
}

enum ProtocolState {
    AwaitingResponse,
    Completed,
    Failed,
}

impl PairingProtocol {
    pub fn initiate(seed: &SecureSeed, device_pk: &[u8; 32]) -> Result<(Self, Vec<u8>), ProtocolError>;
    pub fn complete(self, response_bytes: &[u8]) -> Result<CompletedPairing, ProtocolError>;
}

pub struct CompletedPairing {
    pub shared_secret: [u8; 32],
    pub peer_public_key: [u8; 32],
}
```

**Key constraint:** `EphemeralSecret` from x25519_dalek is `!Clone + !Serialize`. The state machine is inherently ephemeral — it lives in memory only. Document this. Mobile apps must complete pairing in one session.

## Add protocol state machine and update publish ordering

### Protocol state machine

Add a formal state machine to `auths-pairing-protocol` that makes the pairing protocol lifecycle explicit.

```rust
/// Transport-agnostic pairing protocol state machine.
///
/// Usage:
/// ```ignore
/// let (state, token_bytes) = PairingProtocol::initiate(seed, device_pk)?;
/// // Caller sends token_bytes over their transport (HTTP, BLE, etc.)
/// // Caller receives response_bytes from peer
/// let result = state.complete(response_bytes)?;
/// ```
pub enum PairingProtocol {
    /// Session initiated, token generated, awaiting peer response.
    AwaitingResponse {
        session: PairingSession,
        // token metadata for correlation
    },
    /// Exchange completed successfully.
    Completed {
        shared_secret: SharedSecret,
        peer_public_key: [u8; 32],
    },
    /// Exchange failed.
    Failed {
        error: ProtocolError,
    },
}
```

### State transitions

```
Initiated → AwaitingResponse (via initiate())
AwaitingResponse → Completed (via complete())
AwaitingResponse → Failed (via complete() with bad response)
```

### Key constraint

`EphemeralSecret` from x25519_dalek is `!Clone + !Serialize`. The state machine is inherently ephemeral — it lives in memory only and cannot be persisted across app restarts. Document this clearly.
