# fn-13.3 Implement CLI agent adapter and refactor bin/sign.rs

## Description
Implement a CLI adapter for `AgentSigningPort` that wraps the existing Unix-only agent client. Refactor `bin/sign.rs::run_sign()` to delegate to `CommitSigningWorkflow`.

### What to do

1. **Create `crates/auths-cli/src/adapters/agent.rs`**:
   - `CliAgentAdapter` struct implementing `AgentSigningPort`
   - `try_sign()`: wraps `auths_core::agent::client::agent_sign()` (returns raw bytes) + `construct_sshsig_pem()` to produce SSHSIG PEM string matching `sign_with_seed()` format
   - `ensure_running()`: wraps `ensure_agent_running(true)` from `commands/agent.rs` — the `quiet=true` parameter suppresses all stderr output, so the adapter is safe to use silently
   - `add_identity()`: wraps `auths_core::agent::client::add_identity()` passing PKCS#8 DER bytes
   - Gate with `#[cfg(unix)]` — on non-Unix, this module doesn't exist; CLI wires `NoopAgentProvider`
   - Register in `crates/auths-cli/src/adapters/mod.rs`

2. **Update CLI factory** (`crates/auths-cli/src/factories/mod.rs`):
   - Wire `CliAgentAdapter` into `AuthsContext` builder on Unix
   - Wire `NoopAgentProvider` on non-Unix platforms

3. **Refactor `crates/auths-cli/src/bin/sign.rs`**:
   - Replace `run_sign()` body (L314-378) with call to `CommitSigningWorkflow::execute()`
   - Build `CommitSigningParams` from CLI args
   - Pass `chrono::Utc::now()` at the presentation boundary

   **What stays in CLI (explicit enumeration):**
   - Arg parsing (clap) and namespace extraction from CLI args
   - Pubkey resolution: loading from keychain, calling `get_cached_pubkey()` / `extract_pubkey_from_key_bytes()` — passed to `CommitSigningParams.pubkey`
   - `--delegate-to-ssh-keygen` branch (already out of scope)
   - `run_verify()` and `run_delegate_to_ssh_keygen()` functions (unchanged)
   - Output file write: `fs::write(&sig_path, &signature_pem)` after workflow returns
   - `print!("{pubkey_b64}")` stdout output for Git's allowed signers

   **What moves to workflow:**
   - Three-tier signing logic (agent → auto-start+direct → direct)
   - Passphrase retry loop (was `load_key_with_retry`)
   - `validate_freeze_state()` call
   - `sign_with_seed()` call

   **What gets removed from CLI:**
   - `process::exit(1)` — replaced by `SigningError::PassphraseExhausted`
   - `anyhow!("{}", e)` patterns — preserve typed `SigningError`
   - Direct calls to `try_sign_via_agent()`, `ensure_agent_running()`, `add_identity()`

4. **Update `crates/auths-cli/src/errors/renderer.rs`**:
   - Add downcast handling for new `SigningError` variants
   - Map `PassphraseExhausted` to user-friendly message with suggestion
