# fn-12.5 auths-id: add test-utils feature

## Description

Add a `test-utils` feature to `auths-id` that exposes fakes, contract test macros, fixtures, and mocks for traits defined in `auths-id`.

### What to move

**Fakes** (from `crates/auths-test-utils/src/fakes/`):
- `FakeRegistryBackend` (implements `RegistryBackend` from `auths-id/src/storage/registry/backend.rs:293`)
- `FakeAttestationSink` (implements `AttestationSink` from `auths-id/src/attestation/export.rs:17`)
- `FakeAttestationSource` (implements `AttestationSource` from `auths-id/src/storage/attestation.rs:31`)
- `FakeIdentityStorage` (implements `IdentityStorage` from `auths-id/src/storage/identity.rs:34`)

**Contract test macros** (from `crates/auths-test-utils/src/contracts/`):
- `registry_backend_contract_tests!` — rewrite all `auths_test_utils::fixtures::*` references to `$crate::testing::fixtures::*`
- Delete `session_store_contract_tests!` — references `auths_auth_server` which is not a workspace member (dead code)

**Fixtures** (from `crates/auths-test-utils/src/fixtures/`):
- `test_inception_event(key_seed: &str) -> Event`
- `test_attestation(device_did, issuer) -> Attestation`

**Mocks** (from `crates/auths-test-utils/src/mocks/`):
- `MockIdentityStorage` (mockall `#[automock]`)
- `MockAttestationSource` (mockall `#[automock]`)

### Where to put it

Create `crates/auths-id/src/testing/mod.rs` with submodules:
```
testing/
  mod.rs
  fakes/
    mod.rs
    registry.rs
    attestation.rs
    identity_storage.rs
  contracts/
    mod.rs
    registry.rs
  fixtures/
    mod.rs
  mocks/
    mod.rs
```

Gate in `crates/auths-id/src/lib.rs`:
```rust
#[cfg(any(test, feature = "test-utils"))]
pub mod testing;
```

### Important: git-storage gating

The traits `AttestationSource`, `AttestationSink`, `IdentityStorage` are currently behind `#[cfg(feature = "git-storage")]` in `auths-id`. Since `git-storage` is the default feature, the fakes compile in normal builds. The `test-utils` feature should temporarily also enable `git-storage` until Phase 2 (fn-12.10) ungates the trait definitions.

### Important: Clock injection fix

`FakeRegistryBackend.metadata()` currently calls `chrono::Utc::now()`. This violates the clock injection rule (CLAUDE.md). Fix during migration: accept `now: DateTime<Utc>` parameter or use a fixed test timestamp.

### Important: Contract macro `$crate::` paths

`registry_backend_contract_tests!` macro body has 14+ references to `auths_test_utils::fixtures::*`. These MUST become `$crate::testing::fixtures::*`. Remember: `#[macro_export]` places the macro at the `auths_id` crate root. All `$crate::testing::fixtures::*` types must be `pub`.

### Cargo.toml changes

```toml
[features]
test-utils = [
    "git-storage",            # temporary, until fn-12.10 ungates traits
    "auths-crypto/test-utils",
    "dep:mockall",
    "dep:rand",
]
```

Add `mockall` and `rand` as optional normal dependencies (NOT dev-dependencies, since downstream crates need them via the feature flag).

### Files to modify

- `crates/auths-id/Cargo.toml` — add features, optional deps
- `crates/auths-id/src/lib.rs` — add gated `testing` module
- Create `crates/auths-id/src/testing/**` — all moved helpers
