## Create FFI-safe facade types and RegistryBackend adapter

### Context

`AuthsContext` at `crates/auths-sdk/src/context.rs:92-114` has 9 `Arc<dyn Trait>` fields that cannot cross the FFI boundary. `RegistryBackend` at `crates/auths-id/src/storage/registry/backend.rs:293-379` uses `&mut dyn FnMut` visitors and `ControlFlow` ‚Äî incompatible with uniffi.

Mobile apps need a concrete, uniffi-exportable facade that hides the trait-object complexity.

### What to Do

1. Create a `SimpleRegistryAdapter` struct (in a new module within the FFI crate or in auths-id) that wraps `RegistryBackend` and replaces visitor methods with `Vec<T>` returns:
   - `visit_events(visitor)` ‚Üí `list_events() -> Vec<Event>`
   - `visit_devices(visitor)` ‚Üí `list_devices() -> Vec<DeviceInfo>`
   - `visit_attestation_history(visitor)` ‚Üí `list_attestation_history() -> Vec<Attestation>`
   - `visit_org_member_attestations(visitor)` ‚Üí `list_org_member_attestations() -> Vec<Attestation>`
   - `visit_identities(visitor)` ‚Üí `list_identities() -> Vec<IdentityInfo>`

2. Create `MobileContext` ‚Äî a concrete struct annotated with `#[derive(uniffi::Object)]` that:
   - Holds concrete implementations internally (not trait objects)
   - Provides a constructor that accepts configuration (registry URL, etc.) and callback interfaces (keychain, passphrase)
   - Internally constructs an `AuthsContext` with the provided implementations
   - Exposes key SDK methods as uniffi-exported methods

3. Create `MobileVerifier` ‚Äî wraps the `Verifier` struct with sync verification methods:
   - `verify_attestation(json: Vec<u8>, pk: Vec<u8>) -> Result<VerificationReport, VerifierError>`
   - `verify_chain(chain_json: Vec<u8>, root_pk: Vec<u8>) -> Result<ChainReport, VerifierError>`

4. Define uniffi-compatible data types (`#[derive(uniffi::Record)]`) for all types that cross the boundary:
   - `VerificationReport`, `ChainReport`, `AttestationInfo`, `DeviceInfo`, `IdentityInfo`
   - Use only primitive types + String + Vec<u8> + Option<T> + Vec<T>

### Key Files

- `crates/auths-sdk/src/context.rs:92-114` ‚Äî AuthsContext with 9 Arc<dyn> fields
- `crates/auths-id/src/storage/registry/backend.rs:293-379` ‚Äî RegistryBackend with FnMut visitors
- `crates/auths-verifier/src/verifier.rs:28-42` ‚Äî Verifier struct (clean DI container)
- `packages/auths-verifier-swift/src/lib.rs` ‚Äî existing uniffi Record/Enum patterns to follow

### Constraints

- Facade types must be `Send + Sync` (uniffi requirement)
- No `&mut self` methods on uniffi Objects (use interior mutability with Mutex if needed)
- All data types must use uniffi-compatible primitives only
- Do not modify the existing `RegistryBackend` trait ‚Äî the adapter wraps it
- Do not modify `AuthsContext` ‚Äî the facade constructs it internally
