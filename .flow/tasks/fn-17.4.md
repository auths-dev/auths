## Integration tests for trust-registry-gated token exchange

End-to-end tests verifying the trust registry gates token exchange correctly.

### Test location

`crates/auths-oidc-bridge/tests/cases/trust_registry.rs` ŌĆö feature-gated behind `#[cfg(feature = "oidc-trust")]`.

### Test cases

1. **Trusted provider, allowed repo, matching caps** ŌĆö exchange succeeds, JWT issued with intersected capabilities

2. **Untrusted provider** ŌĆö exchange returns `ProviderNotTrusted` (403)

3. **Trusted provider, disallowed repo** ŌĆö exchange returns `RepositoryNotAllowed` (403)

4. **Trusted provider, no capability overlap** ŌĆö exchange returns `CapabilityNotAllowed` (403)

5. **TTL enforcement** ŌĆö request TTL of 7200s, registry max 3600s ŌåÆ issued JWT has 3600s TTL

6. **No trust registry configured** ŌĆö exchange succeeds (backwards compatible)

7. **Trust registry + policy evaluation combined** ŌĆö enable both `oidc-trust` and `oidc-policy` features. Both trust check and policy must pass for JWT issuance. Verify ordering: trust check failure returns trust error (e.g. `ProviderNotTrusted`), not policy error (`PolicyDenied`). Feature-gate this test behind `#[cfg(all(feature = "oidc-trust", feature = "oidc-policy"))]`.

### Helpers

- Create temp trust registry JSON files for test fixtures (use `tempfile` crate)
- Reuse keypair and attestation chain helpers from `auths-test-utils`
- Use `PolicyBuilder` for constructing test policies (test 7 only)

### Error assertions

Tests should assert on the specific `BridgeError` variant (pattern match), not just HTTP status codes. This ensures the correct error path is hit.
