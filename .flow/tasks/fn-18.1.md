## Add schema feature flag and JsonSchema derives to contract types

Add schemars 0.8 as a workspace dependency and add `JsonSchema` derives to the owned contract types that need JSON Schema generation.

### Workspace dependency

Add to `Cargo.toml` workspace dependencies:
```toml

```

auths-core already uses schemars 0.8 for pairing types. Standardize it as a workspace dep so all crates use the same version.

[dependencies]

2. **IdentityBundle** â€” if defined in auths-id. If it's only in auths-verifier, create a mirror type in auths-id for schema generation.
