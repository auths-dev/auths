## Integrate trust registry checks into bridge admission flow

Wire trust registry checks into `OidcIssuer::exchange()` as a pre-filter before policy evaluation.

### Integration point

`crates/auths-oidc-bridge/src/issuer.rs` — `OidcIssuer::exchange()`.

Insert trust check BEFORE the policy evaluation added in fn-16.3:

```
1. Decode root public key
2. Validate audience
3. Validate TTL
4. Verify attestation chain
5. Check validity window
6. *** TRUST REGISTRY CHECK (new) ***
7. *** POLICY EVALUATION (fn-16.3) ***
8. Extract capabilities (intersected with trust entry's allowed caps)
9. Build JWT claims
10. Sign JWT
```

### Trust check logic

```rust
#[cfg(feature = "oidc-trust")]
let (effective_caps, effective_ttl) = if let Some(ref registry) = trust_registry {
    let provider_issuer = determine_provider_issuer(&request);
    let entry = registry.lookup(&provider_issuer)
        .ok_or_else(|| BridgeError::ProviderNotTrusted {
            provider: provider_issuer,
        })?;

    if let Some(ref repo) = github_claims_repo {
        if !entry.repo_allowed(repo) {
            return Err(BridgeError::RepositoryNotAllowed {
                repo: repo.clone(),
                provider: entry.provider_issuer.to_string(),
            });
        }
    }

    let effective_caps = entry.intersect_capabilities(&requested_caps);
    if effective_caps.is_empty() {
        return Err(BridgeError::CapabilityNotAllowed {
            requested: requested_caps.iter().map(|c| c.to_string()).collect(),
            allowed: entry.allowed_capabilities.iter().map(|c| c.to_string()).collect(),
        });
    }

    let effective_ttl = std::cmp::min(request_ttl, entry.max_token_ttl_seconds);
    (effective_caps, effective_ttl)
} else {
    (requested_caps, request_ttl)
};
```

### New BridgeError variants (feature-gated behind `oidc-trust`)

Use structured fields, not raw Strings:

```rust
#[cfg(feature = "oidc-trust")]
#[error("OIDC provider not in trust registry: {provider}")]
ProviderNotTrusted { provider: String },

#[cfg(feature = "oidc-trust")]
#[error("repository not allowed for provider {provider}: {repo}")]
RepositoryNotAllowed { repo: String, provider: String },

#[cfg(feature = "oidc-trust")]
#[error("no allowed capabilities match request")]
CapabilityNotAllowed { requested: Vec<String>, allowed: Vec<String> },
```

Map all three to HTTP 403 in `IntoResponse`.

### Capability intersection — compose, don't duplicate

The existing `scope_capabilities()` in `issuer.rs` does `requested ∩ chain_granted`. Trust registry adds a third set: `registry_allowed`. Rather than duplicating intersection logic, compose them:

1. Trust registry check: `effective_caps = requested ∩ registry_allowed` (via `entry.intersect_capabilities()`)
2. Existing scope check: `final_caps = effective_caps ∩ chain_granted` (via existing `scope_capabilities()`)

This means the trust registry narrows the requested set BEFORE it hits the chain-granted intersection. Three-way intersection without duplicating any logic.

### Fail-safe defaults

- No trust registry configured (`None`): allow all (backwards compatible)
- Trust registry configured but no matching provider: **deny** (fail-closed)
