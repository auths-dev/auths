## Feature-gate tokio in auths-core

### Context

`auths-core` has `tokio = { version = "1", features = ["full"] }` as a hard dependency (`Cargo.toml:29`). This pulls the full tokio runtime into any crate that depends on auths-core, including mobile FFI. On iOS this adds ~1-3MB binary size for functionality mobile doesn't use. The `provider_bridge.rs` pattern (`FALLBACK_RT`, `block_in_place`) requires tokio at runtime even for sync operations.

### What to Do

1. Create a feature flag `async-runtime` (default = on) in `crates/auths-core/Cargo.toml` that gates the tokio dependency.

2. Gate `crates/auths-core/src/crypto/provider_bridge.rs` behind `#[cfg(feature = "async-runtime")]`. When the feature is off, the bridge module doesn't exist — consumers use `SyncCryptoProvider` directly (from fn-11.1).

3. Gate other tokio-dependent code in auths-core:
   - `crates/auths-core/src/storage/linux_secret_service.rs` — uses `tokio::task::block_in_place`
   - `crates/auths-core/src/storage/passphrase_cache.rs:364-459` — uses `tokio::task::block_in_place`
   - `crates/auths-core/src/witness/` — witness server (already behind `witness-server` feature)
   - `crates/auths-core/src/agent/session.rs` — SSH agent (async)

4. Ensure `cargo check -p auths-core --no-default-features` compiles without tokio.

5. All existing consumers (auths-sdk, auths-cli, auths-id) continue working with the default `async-runtime` feature on.

### Key Files

- `crates/auths-core/Cargo.toml:29` — tokio hard dependency
- `crates/auths-core/src/crypto/provider_bridge.rs` — FALLBACK_RT, block_in_place
- `crates/auths-core/src/storage/linux_secret_service.rs` — tokio block_in_place
- `crates/auths-core/src/storage/passphrase_cache.rs` — tokio block_in_place
- `crates/auths-core/src/witness/` — witness server (already feature-gated)
- `crates/auths-core/src/agent/` — SSH agent

### Constraints

- Default features must include `async-runtime` — no breakage for existing consumers
- Linux secret service storage should be gated behind `async-runtime` (it requires tokio)
- Passphrase cache macOS implementation that uses block_in_place needs conditional compilation
- `cargo nextest run -p auths-core` must still pass with default features
