## Define trust registry types and matching logic in auths-policy

Create `crates/auths-policy/src/trust.rs` with trust registry types and matching logic. All dependencies (`ValidatedGlob`, `CanonicalCapability`, `glob_match`) are already in `auths-policy`.

### Types

```rust
/// A validated OIDC issuer URL (HTTPS, no wildcards, no path traversal).
#[derive(Debug, Clone, PartialEq, Eq, Hash, Serialize, Deserialize)]
#[serde(try_from = "String", into = "String")]
pub struct ValidatedIssuerUrl(String);

#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub struct TrustRegistryEntry {
    pub provider_issuer: ValidatedIssuerUrl,
    pub allowed_repos: Vec<ValidatedGlob>,
    pub allowed_capabilities: Vec<CanonicalCapability>,
    pub max_token_ttl_seconds: u64,
    pub require_witness_quorum: Option<usize>,
}

#[derive(Debug, Clone, Default, PartialEq, Eq, Serialize, Deserialize)]
pub struct TrustRegistry {
    entries: Vec<TrustRegistryEntry>,
}
```

### ValidatedIssuerUrl

Newtype for OIDC provider issuer URLs. Validation:
- Must start with `https://` (no HTTP, no wildcards)
- Must be valid URL structure (has host)
- No `..` path traversal
- Max 512 chars
- Stored as-is (no normalization beyond trimming)

Implementation note: `auths-policy` doesn't have a `url` crate dependency and shouldn't add one for this. Use string-based validation: check `https://` prefix, check for host segment after `://`, reject control chars. This matches the minimal-dependency philosophy of the policy crate.

### Matching logic — methods on the correct types

```rust
impl TrustRegistry {
    /// Look up the trust entry for a provider. Returns None if not in registry.
    pub fn lookup(&self, provider_issuer: &str) -> Option<&TrustRegistryEntry>;
}

impl TrustRegistryEntry {
    /// Check if a repo is allowed by this entry's repo patterns.
    /// First segment (org) must be literal — no wildcards on org name.
    pub fn repo_allowed(&self, repo: &str) -> bool;

    /// Intersect requested capabilities with this entry's allowed capabilities.
    /// Returns the effective (narrowed) set.
    pub fn intersect_capabilities(
        &self,
        requested: &[CanonicalCapability],
    ) -> Vec<CanonicalCapability>;
}
```

`repo_allowed` and `intersect_capabilities` are methods on `TrustRegistryEntry`, not `TrustRegistry` — they operate on a single entry.

### Validation

- `ValidatedIssuerUrl` constructor validates HTTPS, no wildcards, valid structure
- `TrustRegistryEntry` validates via serde `try_from`:
  - `allowed_repos` patterns must have literal first segment (org)
  - `max_token_ttl_seconds > 0`
- Multiple entries for the same `provider_issuer` are NOT merged — first match wins

### Serde

Use `#[serde(try_from = "...")]` pattern for `ValidatedIssuerUrl`, matching the existing `ValidatedGlob`/`CanonicalDid` approach.
