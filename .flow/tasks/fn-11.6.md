## Unify FFI crates and upgrade to uniffi 0.31

### Context

Currently there are two separate FFI crates:
- `crates/auths-mobile-ffi/` ‚Äî identity creation, signing, pairing (uniffi 0.28)
- `packages/auths-verifier-swift/` ‚Äî verification only (uniffi 0.28)

Both are excluded from the workspace, pin dependencies independently, and have no CI coverage. If a mobile app needs both identity AND verification, it must link two separate libraries with duplicate `ring`/`blake3` symbols ‚Äî risk of link-time conflicts.

### What to Do

1. Create a unified `crates/auths-ffi/` crate that combines identity, verification, signing, and pairing functionality.

2. Upgrade to uniffi 0.31 (from 0.28). Key migration steps:
   - `UniffiCustomTypeConverter` ‚Üí `custom_type!` macro (breaking change in 0.29)
   - Test generated bindings compile with Swift 5.9+ and Kotlin 1.9+

3. Set up the crate:
   - `Cargo.toml`: `crate-type = ["lib", "staticlib", "cdylib"]`, own `[workspace]`
   - `uniffi::setup_scaffolding!()` in lib.rs
   - `uniffi.toml` for Swift and Kotlin binding configuration
   - `uniffi-bindgen.rs` binary for generating bindings

4. Migrate existing types from both crates:
   - From `auths-mobile-ffi`: `IdentityResult`, `PairingInfo`, `PairingResponsePayload`, `MobileError`, all exported functions
   - From `auths-verifier-swift`: `UniVerificationReport`, `UniChainLink`, `UniAttestationError`, all exported functions

5. Integrate facade types from fn-11.4 and callback interfaces from fn-11.5.

6. Update build scripts:
   - `build-swift.sh` ‚Üí points to unified crate
   - `build-xcframework.sh` ‚Üí builds unified XCFramework
   - `build-android.sh` ‚Üí builds unified .so/.aar

7. Deprecate (but don't delete yet) old crates with README noting migration to `auths-ffi`.

### Key Files

- `crates/auths-mobile-ffi/Cargo.toml` ‚Äî current mobile FFI crate
- `crates/auths-mobile-ffi/src/lib.rs` ‚Äî current mobile FFI code
- `packages/auths-verifier-swift/Cargo.toml` ‚Äî current Swift verifier
- `packages/auths-verifier-swift/src/lib.rs` ‚Äî current Swift verifier code
- `packages/auths-verifier-swift/build-swift.sh` ‚Äî Swift build script
- `packages/auths-verifier-swift/build-xcframework.sh` ‚Äî XCFramework build
- `packages/auths-verifier-swift/build-android.sh` ‚Äî Android build

### Constraints

- Unified crate must remain excluded from the main workspace (own `[workspace]`)
- Single cdylib ‚Äî no symbol duplication
- All existing Swift/Kotlin API consumers must have a migration path (type renames documented)
- uniffi 0.31 proc-macro approach only (no UDL files)
- release profile: `strip = "symbols"`, `lto = true`, `opt-level = "z"`, `panic = "abort"`
