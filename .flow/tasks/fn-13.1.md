# fn-13.1 Define AgentSigningPort trait and add to AuthsContext

## Description
Define an `AgentSigningPort` trait in `crates/auths-sdk/src/ports/agent.rs` that abstracts the agent-based signing operations. Add this as an optional field on `AuthsContext` with a `NoopAgentProvider` default.

### What to do

1. **Create `crates/auths-sdk/src/ports/agent.rs`** with:
   - `AgentSigningPort` trait (`Send + Sync + 'static`) with methods:
     - `try_sign(&self, namespace: &str, pubkey: &[u8], data: &[u8]) -> Result<String, AgentSigningError>` — attempt signing via running agent. **Returns SSHSIG PEM string** (the `-----BEGIN SSH SIGNATURE-----` format). The adapter is responsible for converting raw Ed25519 bytes from the agent wire protocol into SSHSIG PEM using `construct_sshsig_pem()`. This matches the return format of `sign_with_seed()` so the workflow doesn't need to normalize.
     - `ensure_running(&self) -> Result<(), AgentSigningError>` — start agent if not running (called with quiet=true semantics — no stdout/stderr output)
     - `add_identity(&self, namespace: &str, pkcs8_der: &[u8]) -> Result<(), AgentSigningError>` — load a decrypted PKCS#8 DER-encoded keypair into the running agent. The bytes must be the output of keychain decryption (i.e. `decrypt_keypair()` result), not raw seed or encrypted blob.
   - `AgentSigningError` thiserror enum with variants: `Unavailable(String)`, `ConnectionFailed(String)`, `SigningFailed(String)`, `StartupFailed(String)`
   - `NoopAgentProvider` struct implementing `AgentSigningPort` — all methods return `Err(AgentSigningError::Unavailable("agent not supported on this platform"))`

2. **Register in `crates/auths-sdk/src/ports/mod.rs`** — add `pub mod agent;`

3. **Add to `AuthsContext`** (`crates/auths-sdk/src/context.rs`):
   - New field: `pub agent_signing: Arc<dyn AgentSigningPort + Send + Sync>`
   - Builder gets `.agent_signing(impl AgentSigningPort)` method
   - Default to `NoopAgentProvider` if not set (optional field, like `event_sink`)

4. **Create fake** in `crates/auths-sdk/src/testing/fakes/agent.rs`:
   - `FakeAgentProvider` with configurable responses via `Arc<Mutex<Vec<...>>>`
   - Register in `testing/fakes/mod.rs`
   - Gate behind `#[cfg(any(test, feature = "test-utils"))]`

5. **Update test helpers** in `crates/auths-sdk/tests/cases/helpers.rs`:
   - `build_test_context()` should set agent_signing to `FakeAgentProvider` or `NoopAgentProvider`
