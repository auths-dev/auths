# fn-11.5 Add uniffi callback_interfaces for platform traits

## Description
## Add uniffi callback_interfaces for platform traits

### Context

Mobile platforms must provide their own implementations for:
- **Keychain/Keystore**: iOS uses Security.framework, Android uses Keystore (JVM-only, no NDK API)
- **Passphrase prompts**: Mobile apps show native UI (biometric, PIN, custom dialog)

The existing `CallbackPassphraseProvider` at `crates/auths-core/src/signing.rs:324-347` was designed for GUI/FFI use but uses a Rust `Box<dyn Fn>` — not uniffi-compatible.

### What to Do

1. Define a `#[uniffi::export(with_foreign)]` trait for secure key storage:
   ```rust
   #[uniffi::export(with_foreign)]
   pub trait MobileKeyStorage: Send + Sync {
       fn store_key(&self, identifier: String, key_data: Vec<u8>) -> Result<(), MobileError>;
       fn load_key(&self, identifier: String) -> Result<Vec<u8>, MobileError>;
       fn delete_key(&self, identifier: String) -> Result<(), MobileError>;
       fn has_key(&self, identifier: String) -> Result<bool, MobileError>;
   }
   ```
   Swift implements this using iOS Keychain Services. Kotlin implements using Android Keystore.

2. Define a `#[uniffi::export(with_foreign)]` trait for passphrase/biometric prompts:
   ```rust
   #[uniffi::export(with_foreign)]
   pub trait MobilePassphraseProvider: Send + Sync {
       fn get_passphrase(&self, prompt: String) -> Result<String, MobileError>;
   }
   ```

3. Create Rust-side adapter structs that implement `KeyStorage` and `PassphraseProvider` (the core traits) by delegating to the uniffi callback interfaces. These adapters are what gets injected into `AuthsContext`.

4. Wire the adapters into `MobileContext` construction (from fn-11.4):
   - `MobileContext::new(config, key_storage, passphrase_provider)` accepts the callback interfaces

### Key Files

- `crates/auths-core/src/storage/keychain.rs:152-181` — KeyStorage trait (7 methods)
- `crates/auths-core/src/signing.rs:158-179` — PassphraseProvider trait
- `crates/auths-core/src/signing.rs:324-347` — CallbackPassphraseProvider (GUI/FFI design)
- `crates/auths-core/src/storage/ios_keychain.rs` — existing iOS keychain impl (direct Security.framework)
- `crates/auths-core/src/storage/android_keystore.rs` — Android stub (returns errors)

### Constraints

- Callback interface methods must use only uniffi-compatible types (String, Vec<u8>, bool, Result)
- No `Zeroizing<T>` in callback interface signatures (Swift/Kotlin can't express this)
- The adapter from MobileKeyStorage → KeyStorage must handle the type conversion
- All callback interface types must be `Send + Sync` (uniffi ADR-0004)
- Document that mobile implementations MUST handle thread safety (callbacks can come from any thread)
## Acceptance
