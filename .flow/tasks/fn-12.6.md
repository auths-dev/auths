# fn-12.6 auths-sdk: add test-utils feature

## Description

Add a `test-utils` feature to `auths-sdk` that exposes fakes and contract test macros for traits defined in `auths-sdk`.

### What to move

**Fakes** (from `crates/auths-test-utils/src/fakes/`):
- `FakeGitLogProvider` (implements `GitLogProvider` from `auths-sdk/src/ports/git.rs:106`)
- `FakeGitDiagnosticProvider` (implements `GitDiagnosticProvider` from `auths-sdk/src/ports/diagnostics.rs:60`)
- `FakeCryptoDiagnosticProvider` (implements `CryptoDiagnosticProvider` from `auths-sdk/src/ports/diagnostics.rs:74`)

**Contract test macros** (from `crates/auths-test-utils/src/contracts/`):
- `git_log_provider_contract_tests!` — rewrite `auths_sdk::ports::git::GitLogProvider` to `$crate::ports::git::GitLogProvider`

### Why this is a separate task from fn-12.5

These fakes implement traits from `auths-sdk` (Layer 3), NOT from `auths-id` (Layer 1). Placing them in `auths-id` would create an upward dependency violation. They belong in `auths-sdk/src/testing/`.

### Where to put it

Create `crates/auths-sdk/src/testing/mod.rs`:
```
testing/
  mod.rs
  fakes/
    mod.rs
    git.rs
    diagnostics.rs
  contracts/
    mod.rs
    git_log.rs
```

Gate in `crates/auths-sdk/src/lib.rs`:
```rust
#[cfg(any(test, feature = "test-utils"))]
pub mod testing;
```

### Cargo.toml changes

```toml
[features]
test-utils = ["auths-id/test-utils"]  # chain to get auths-id fakes too
```

### Files to modify

- `crates/auths-sdk/Cargo.toml` — add `test-utils` feature
- `crates/auths-sdk/src/lib.rs` — add gated `testing` module
- Create `crates/auths-sdk/src/testing/**` — moved helpers

### Verification

```bash

cargo build -p auths-sdk --features test-utils
cargo nextest run -p auths-sdk
```
