## Wire auths-core pairing module as re-export facade

Convert `auths-core/src/pairing/mod.rs` from containing protocol logic to being a thin re-export facade over `auths-pairing-protocol`.

### Re-export facade

```rust
// crates/auths-core/src/pairing/mod.rs

// Re-export all protocol types
pub use auths_pairing_protocol::{
    PairingToken, PairingSession, PairingResponse,
    ProtocolError,
    // all types from types.rs
    PairingUri, ShortCode, normalize_short_code,
    // etc.
};

// Local modules that stay in auths-core
mod qr;
pub use qr::{QrOptions, format_pairing_qr, render_qr};

// Local error type wrapping protocol + transport errors
mod error;
pub use error::PairingError;
```

### auths-core error wrapper

Create a new `PairingError` in auths-core that wraps `ProtocolError` and adds transport variants:

```rust
#[derive(Debug, thiserror::Error)]
pub enum PairingError {
    #[error(transparent)]
    Protocol(#[from] auths_pairing_protocol::ProtocolError),

    #[error("relay error: {0}")]
    RelayError(String),
    #[error("local server error: {0}")]
    LocalServerError(String),
    #[error("mDNS error: {0}")]
    MdnsError(String),
    #[error("no peer found")]
    NoPeerFound,
    #[error("LAN timeout")]
    LanTimeout,
    #[error("QR code generation failed: {0}")]
    QrCodeFailed(String),
}
```
