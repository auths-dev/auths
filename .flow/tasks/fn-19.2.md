## Add concurrent batch writer tests

Integration tests verifying that concurrent batch writers don't corrupt the registry.

### Test location

`crates/auths-storage/tests/cases/concurrent_batch.rs` — add as new submodule.

### Test cases

1. **Sequential batches** — two batches applied sequentially produce correct state
2. **Parallel batch writers** — spawn N threads, each submitting a batch. Advisory lock ensures only one succeeds at a time. After all complete, verify:
   - All events are present in the registry
   - No events are duplicated
   - No events are lost
   - Metadata counters are correct
3. **Batch + single-event interleaving** — one thread does `batch_append_events`, another does `append_event`. Both complete successfully without corruption.
4. **CAS retry under contention** — if batch CAS fails (another writer committed between lock release and CAS check), verify the error is clear and the caller can retry.

### Helpers

Reuse patterns from existing `concurrent_writes.rs`:
- Thread pool with barriers for synchronized start
- Retry loop with bounded attempts
- Post-test validation scanning all events
