## Integration tests for policy-gated token exchange

Write integration tests verifying the end-to-end flow of policy-gated token exchange.

### Test location

`crates/auths-oidc-bridge/tests/cases/policy_gate.rs` — add as a new submodule. Feature-gate the entire module behind `#[cfg(feature = "oidc-policy")]`.

### Test cases

1. **Exchange with allowing policy** — build a policy that allows `SignerType::Workload` with `sign:commit` capability. Create an exchange request with matching chain. Verify JWT is issued successfully.

2. **Exchange with denying policy** — build a policy that requires `deploy:production` capability. Create an exchange request with only `sign:commit`. Verify exchange returns `PolicyDenied` error (not `InvalidRequest`).

3. **Exchange with no policy configured** — set workload policy to `None` in state. Verify exchange succeeds (backwards compatible).

4. **Exchange with policy checking repo** — build a policy with `RepoIs("myorg/myrepo")`. Create exchange request with matching `github_repository`. Verify success. Then change repo to non-matching. Verify denial.

5. **Exchange with policy checking workload issuer** — build a policy with `WorkloadIssuerIs(did)`. Verify it matches when `keri_prefix` maps correctly.

- Construct `OidcClaims` test fixtures
- Construct `CompiledPolicy` via `PolicyBuilder`
- Use `auths-test-utils` for keypairs and repos
