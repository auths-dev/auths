# fn-12.11 Move git-storage implementations to auths-storage

## Description

Move git-based storage implementations from `auths-id` to `auths-storage`. After fn-12.10 ungated the trait definitions, the git implementations behind `#[cfg(feature = "git-storage")]` can now be moved to `auths-storage` where they belong (infrastructure layer).

### What to move

All code currently gated behind `#[cfg(feature = "git-storage")]` in `auths-id/src/`:
- Git-based `IdentityStorage` implementation
- Git-based `AttestationSource`/`AttestationSink` implementations
- Git ref reading/writing for identity data (`storage/git_refs.rs`)
- KERI git storage operations (`keri/mod.rs` gated sections)
- Local `~/.auths` directory management
- `agent_identity`, `trust`, `witness` gated modules (if they contain git-specific code)

### Where to put it

`crates/auths-storage/src/git/` — `auths-storage` already has a `backend-git` feature with `git2`, `fs2`, `tempfile`, `tokio` deps.

### Cargo.toml changes

- `auths-id/Cargo.toml`: Remove `git-storage` feature entirely. Remove `git2`, `dirs`, `tempfile`, `tokio` from dependencies.
- `auths-storage/Cargo.toml`: May need additional deps that were previously in `auths-id`
- Consumer crates that used `auths-id/git-storage` must switch to `auths-storage/backend-git`

### Files to modify

- `crates/auths-id/Cargo.toml` — remove `git-storage` feature and its deps
- `crates/auths-id/src/` — remove all git-gated code (already split in fn-12.10)
- `crates/auths-storage/src/git/` — receive the moved implementations
- `crates/auths-storage/Cargo.toml` — add any missing deps
- Any consumer crate that enables `auths-id/git-storage`

### Verification

```bash
cargo build -p auths-id
cargo build -p auths-storage --features backend-git
cargo nextest run --workspace
```
