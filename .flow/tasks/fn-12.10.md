# fn-12.10 auths-id: ungate trait definitions from git-storage

## Description

Ungate trait definitions in `auths-id` that are currently behind `#[cfg(feature = "git-storage")]`. Split trait definitions (domain, must stay in `auths-id` ungated) from their git-based implementations (infrastructure, will move to `auths-storage` in fn-12.11).

### Problem

These modules are ENTIRELY gated behind `git-storage`:
- `auths-id/src/storage/attestation.rs` — contains `AttestationSource` trait + git implementation
- `auths-id/src/storage/identity.rs` — contains `IdentityStorage` trait + git implementation
- `auths-id/src/attestation/export.rs` — contains `AttestationSink` trait
- `auths-id/src/identity/helpers.rs` — contains `ManagedIdentity` type
- `auths-id/src/storage/mod.rs` — submodules `attestation`, `git_refs`, `receipts`, `identity`, `keri`
- `auths-id/src/keri/mod.rs` — KERI git storage operations (lines 99-136)

The traits are domain concepts. The git implementations are infrastructure. They must be separated.

### Approach

For each affected module:

1. **Split the file** into trait definition (ungated) + git implementation (stays gated behind `git-storage`)
2. Example for `storage/attestation.rs`:
   - Keep `AttestationSource` trait definition **ungated**
   - Move git-specific `impl AttestationSource for GitAttestationStorage` to a sub-file gated behind `git-storage`
3. Ensure all trait definitions, associated types, and error types are available without `git-storage`
4. `git2` imports must only appear in gated blocks

### After this task

- `auths-id/test-utils` feature no longer needs to enable `git-storage` (update Cargo.toml)
- Trait types like `AttestationSource`, `IdentityStorage`, `ManagedIdentity` are always available

### Key files to modify

- `crates/auths-id/src/storage/mod.rs` — ungate trait submodules
- `crates/auths-id/src/storage/attestation.rs` — split trait from git impl
- `crates/auths-id/src/storage/identity.rs` — split trait from git impl
- `crates/auths-id/src/attestation/mod.rs` — ungate relevant modules
- `crates/auths-id/src/attestation/export.rs` — split trait from git impl
- `crates/auths-id/src/identity/mod.rs` — ungate relevant modules
- `crates/auths-id/src/identity/helpers.rs` — ungate `ManagedIdentity`
- `crates/auths-id/src/keri/mod.rs` — gate only git operations, not KERI logic
- `crates/auths-id/src/error.rs` — ungate trait error variants, keep git-specific errors gated
- `crates/auths-id/Cargo.toml` — remove `git-storage` from `test-utils` feature
