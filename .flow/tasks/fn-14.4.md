## Description
# Task: Version String Computation

Replace the hardcoded `KERI10JSON000000_` with dynamically computed version strings containing the actual hex-encoded byte count of the serialized event body.

## Format

`KERI10JSON{NNNNNN}_` where `{NNNNNN}` is a 6-char lowercase hex byte count. Always exactly 17 characters. Max event size: 16,777,215 bytes (0xFFFFFF).

## Fixed-point algorithm

The version string is part of the body it measures. Because the size field is always exactly 6 hex chars (same width as `000000`), the total byte count is stable in one pass:

1. Serialize event with placeholder `KERI10JSON000000_`
2. Measure byte length
3. Format as `KERI10JSON{size:06x}_`
4. Re-serialize with computed version string
5. Sanity check: final byte count == initial byte count (must be equal since both `000000` and `{size:06x}` are 6 chars)

## What to do

1. Implement `compute_version_string(event: &serde_json::Value) -> Result<(String, Vec<u8>)>`:
   - Uses `serde_json::to_vec` for serialization (consistent with SAID computation)
   - Returns `(version_string, serialized_bytes_with_correct_version)`

2. Write tests:
   - Version string starts with `KERI10JSON` and ends with `_`
   - Always exactly 17 chars
   - Hex size matches actual `serialized_bytes.len()`
   - Different event sizes produce different version strings
