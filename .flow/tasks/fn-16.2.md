## Create OidcClaims-to-EvalContext adapter function

Create a new module `crates/auths-oidc-bridge/src/policy_adapter.rs` (feature-gated behind `oidc-policy`) containing a function that converts `OidcClaims` into `EvalContext`.

### Function signature

```rust
pub fn build_eval_context_from_oidc(
    claims: &OidcClaims,
    now: DateTime<Utc>,
) -> Result<EvalContext, BridgeError>
```

### Field mapping

| OidcClaims field | EvalContext method | Notes |
|------------------|-------------------|-------|
| `keri_prefix` | `.issuer` via `CanonicalDid::parse()` | Root identity that issued the chain |
| `sub` | `.subject` via `CanonicalDid::parse()` | Subject of the JWT |
| `capabilities` | `.capability()` for each | Parse via `CanonicalCapability::parse()`, skip unparseable |
| `github_repository` | `.repo()` | If present |
| `github_actor` | `.workload_claim("github_actor", _)` | If present |
| `target_provider` | `.attr("provider", _)` | If present |
| `keri_prefix` | `.workload_issuer()` | Also set as workload issuer DID |
| (always) | `.signer_type(SignerType::Workload)` | All OIDC contexts are workloads |

### Note on `keri_prefix` vs `sub`

In the current bridge, `sub` is the full DID string (from `chain[0].issuer`) and `keri_prefix` is extracted from it via `extract_keri_prefix()`. For policy evaluation:
- `issuer` = the identity root that issued the attestation chain (derived from `keri_prefix` ŌåÆ `did:keri:{prefix}`)
- `subject` = the JWT subject DID (from `sub`, parsed via `CanonicalDid::parse()`)
- `workload_issuer` = also set from `keri_prefix`, since this is a workload context

If `sub` is already a DID (e.g. `did:keri:EOrg123`), parse directly. If `keri_prefix` is a bare prefix (e.g. `EOrg123`), construct `did:keri:{prefix}` before parsing.

### Pattern to follow

`context_from_attestation()` in `crates/auths-id/src/policy/mod.rs` ŌĆö same approach of converting external claims into `EvalContext`.

### Clock handling

The `now: DateTime<Utc>` parameter is passed in by the caller. The bridge's `ClockFn` (u64 epoch seconds) is converted at the call site in `exchange()`, not in this function.

### Error handling

Return `BridgeError::InvalidRequest` if `CanonicalDid::parse()` fails on `keri_prefix` or `sub`. A DID parse failure is a malformed request, not a policy denial.
