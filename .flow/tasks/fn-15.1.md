# fn-15.1 Consolidate clippy config files

## Description
## Consolidate clippy config files

The workspace has two conflicting clippy config files at the root:
- `clippy.toml` — bans `Utc::now()`, `SystemTime::now()`, `std::env::var`
- `.clippy.toml` — bans `Uuid::new_v4()`, sets `allow-unwrap-in-tests = true` / `allow-expect-in-tests = true`

**Problem**: Clippy reads `.clippy.toml` first (takes precedence). This means the clock/env injection bans in `clippy.toml` may be **silently ignored today**.

### Steps
1. Read both files: `clippy.toml` and `.clippy.toml`
2. Merge all `disallowed-methods` entries into a single `clippy.toml`
3. Include `allow-unwrap-in-tests = true` and `allow-expect-in-tests = true`
4. Delete `.clippy.toml`
5. Verify: `cargo clippy --workspace 2>&1 | grep "disallowed method"` should still catch violations

### Files
- `clippy.toml` (merge target)
- `.clippy.toml` (delete after merge)

### Verification
```bash
# Verify disallowed-methods are enforced
cargo clippy --workspace --all-targets 2>&1 | head -20
# Verify allow-unwrap-in-tests still works
cargo clippy -p auths-core --all-targets 2>&1 | grep -c "unwrap_used"
```
