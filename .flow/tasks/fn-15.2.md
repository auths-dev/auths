# fn-15.2 Migrate lint config to workspace level

## Description
## Migrate lint config to workspace level

Move from per-crate `[lints.clippy]` sections and per-crate `#![deny()]` attributes to `[workspace.lints.clippy]` in root `Cargo.toml`.

### Current state
- 4 crates have `[lints.clippy]` in Cargo.toml (auths-core, auths-crypto, auths-id, auths-keri) — only `print_stdout` and `print_stderr` deny
- 2 crates have `#![deny(clippy::unwrap_used, clippy::expect_used)]` in lib.rs (auths-core, auths-sdk)
- 14+ crates have NO lint configuration at all

### Steps
1. Add to root `Cargo.toml`:
```toml
[workspace.lints.clippy]
unwrap_used = "warn"    # Start as warn, escalate to deny in Task 8
expect_used = "warn"
print_stdout = "deny"
print_stderr = "deny"
```

2. In every member crate's `Cargo.toml`, replace any existing `[lints.clippy]` section with:
```toml
[lints]
workspace = true
```

3. Remove `#![deny(clippy::unwrap_used)]` and `#![deny(clippy::expect_used)]` from `crates/auths-core/src/lib.rs` and `crates/auths-sdk/src/lib.rs` (workspace config covers these now). Keep any other crate-level attributes.

4. Add `#![allow(clippy::unwrap_used, clippy::expect_used)]` to crates that are NOT production code:
   - `crates/auths-test-utils/src/lib.rs` — entire crate is test helpers
   - `xtask/src/main.rs` — build tooling

5. Run `cargo clippy --workspace --all-targets` and note warnings (should be warnings, not errors since we use "warn")

### Files to modify
- `Cargo.toml` (workspace root)
- All member crate `Cargo.toml` files (14+)
- `crates/auths-core/src/lib.rs` (remove deny attributes)
- `crates/auths-sdk/src/lib.rs` (remove deny attributes)
- `crates/auths-test-utils/src/lib.rs` (add allow)
- `xtask/src/main.rs` (add allow)

### Verification
```bash
# Should compile without errors (warnings are OK at this stage)
cargo clippy --workspace --all-targets --all-features 2>&1 | tail -5
# Count warnings to establish baseline
cargo clippy --workspace --all-targets 2>&1 | grep -c "unwrap_used\|expect_used"
```
