## Add #![allow] to integration test entry points and test utility crates

The `allow-unwrap-in-tests` clippy.toml setting only covers `#[cfg(test)]` modules and `#[test]` functions. Integration tests under `tests/` are separate compilation units and need explicit allows.

### Steps
1. Find all `tests/integration.rs` files across the workspace
2. Add `#![allow(clippy::unwrap_used, clippy::expect_used)]` at the top of each
3. Check for any other test entry points (e.g., `tests/*.rs` files that aren't `integration.rs`)
4. Verify `auths-test-utils` already has the allow from Task 2
5. Check `auths-id/src/testing/fixtures.rs` — gated by `#[cfg(any(test, feature = "test-utils"))]`, needs `#[allow]` on the module or functions

### Files
- All `tests/integration.rs` across crates
- `crates/auths-id/src/testing/fixtures.rs` (if test-utils feature triggers lint)

### Verification
```bash
# All integration tests should compile cleanly
cargo clippy --workspace --all-targets -- -W clippy::unwrap_used -W clippy::expect_used 2>&1 | grep "tests/" | head -10
cargo nextest run --workspace
```
