# fn-14.5 Detached-signature event serialization

## Description
# Task: Detached-Signature Event Serialization

Build the event serialization pipeline that strips the `x` field, computes the spec SAID, inserts the correct version string, and produces the signed JSON body ready for CESR attachment.

## CRITICAL: `x` field is base64url, NOT hex

The `x` field contains base64url-no-pad encoded Ed25519 signatures:
- `auths-verifier/src/keri.rs:661-663` -- `URL_SAFE_NO_PAD.decode(sig_str)`
- `auths-id/src/keri/validate.rs:424` -- `URL_SAFE_NO_PAD.decode(sig_str)`

Use `base64::engine::general_purpose::URL_SAFE_NO_PAD` for decode/encode.

## Signing order (critical -- wrong order = unverifiable events)

```
1. Start with auths internal event (has `x` field, has `d` field)
2. Remove `x` field from JSON object
3. Set `d` to 44x `#` placeholder
4. For inception: also set `i` to 44x `#`
5. Serialize with placeholder -> get byte count -> compute version string
6. Re-serialize with version string and placeholders -> Blake3 hash -> SAID
7. Replace `d` placeholder with computed SAID
8. For inception: replace `i` placeholder with computed SAID
9. Re-serialize with final `d`, `i`, and `v` -> this is the signed body
10. The signature is over these final bytes (step 9 output)
11. Attach signature as CESR group AFTER the JSON body
```

## What to do

1. Define `SerializedEvent` struct:
   - `body_bytes: Vec<u8>` -- JSON body with `d`/`i` populated, `x` removed
   - `said: String` -- computed SAID
   - `signature_bytes: Option<Vec<u8>>` -- raw Ed25519 sig (64 bytes) extracted from `x`
   - `signature_key_index: u32` -- 0 for single-sig

2. Implement `serialize_for_cesr(codec, event: &serde_json::Value) -> Result<SerializedEvent>`:
   - Extract `x` field via `URL_SAFE_NO_PAD.decode()` (NOT hex::decode)
   - Re-encode keys in `k` array from internal format to CESR qualified (`D...`)
   - Re-encode commitments in `n` array to CESR qualified (`E...`)
   - Compute spec SAID via `compute_spec_said()`
   - Compute version string via `compute_version_string()`
   - Build final serialized body

3. Implement `decode_cesr_key(codec, qualified: &str) -> Result<Vec<u8>>` for import path

4. Write tests:
   - Output has no `x` field in `body_bytes`
   - Output SAID is 44 chars starting with `E`
   - For inception: `body_bytes` JSON has `d == said` and `i == said`
   - Version string in body has correct hex byte count
   - Keys in `k` array are CESR-qualified (`D...`)
   - `signature_bytes` is `Some(64 bytes)` when input had non-empty `x`

- Given an internal IcpEvent with `x` = base64url signature, output has no `x` field in `body_bytes`
- Output `said` is 44 chars starting with `E`
- Output `body_bytes` parsed as JSON has `d == said` and `i == said` for inception
- Output `body_bytes` parsed as JSON has version string with correct hex byte count
- Keys in `k` array are CESR-qualified (`D...`), not hex
- Commitments in `n` array are CESR-qualified (`E...`), not hex
- `signature_bytes` is `Some(64 bytes)` when input had a non-empty `x` field
- Signature bytes decoded via `URL_SAFE_NO_PAD` (NOT hex)
- All tests pass under `cargo nextest run -p auths-keri -E 'test(event)'`
