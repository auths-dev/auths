## Add #[allow] annotations in auths-verifier and auths-storage

Both crates have a small number of production unwrap/expect calls that are justified. Add `#[allow]` annotations with INVARIANT comments.

### auths-verifier (1 location)

1. **`src/ffi.rs:16`** — `.expect("FFI: failed to create tokio runtime")`
   - **Fix**: `#[allow(clippy::expect_used)]` with `// INVARIANT: Tokio runtime creation is fatal at FFI boundary; cannot propagate Result across FFI`

### auths-storage (~6 locations)

1. **`src/git/adapter.rs:718,980,1152`** — `index.lock().unwrap()` on Mutex
   - **Fix**: `#[allow(clippy::unwrap_used)]` with `// INVARIANT: Mutex poisoning is fatal by design` (matches auths-core convention at `auths-core/src/config.rs:16`)

2. **`src/git/adapter.rs:1312`** — another Mutex lock
   - **Fix**: Same as above

3. **`src/git/adapter.rs:1646`** — `base_tree.clone().unwrap()` after control flow guarantees Some
   - **Fix**: Refactor to `let-else` pattern: `let Some(tree) = base_tree else { ... }`

4. **`src/git/standalone_export.rs:125`** — `.unwrap()` on `previous_attestation` after `is_none()` check
   - **Fix**: Refactor to `if let Some(prev) = previous_attestation { ... }`
