# fn-12.1 auths-crypto: add test-utils feature

## Description

Add a `test-utils` feature to `auths-crypto` that exposes crypto test helpers currently in `auths-test-utils/src/crypto.rs`.

### What to move

From `crates/auths-test-utils/src/crypto.rs`:
- `create_test_keypair(seed: &[u8; 32]) -> (Ed25519KeyPair, [u8; 32])`
- `get_shared_keypair() -> &'static [u8]` (OnceLock pattern)
- `gen_keypair() -> Ed25519KeyPair` (random seed)

### Where to put it

Create `crates/auths-crypto/src/testing.rs` with the moved functions.

Gate in `crates/auths-crypto/src/lib.rs`:
```rust
#[cfg(any(test, feature = "test-utils"))]
pub mod testing;
```

### Cargo.toml changes

```toml
# crates/auths-crypto/Cargo.toml
[features]
test-utils = ["dep:ring"]  # ring is already optional (native feature)
```

Note: `ring` is already an optional dependency for the `native` feature. The `test-utils` feature also activates it. This is fine — features are additive.

### Files to modify

- `crates/auths-crypto/Cargo.toml` — add `test-utils` feature
- `crates/auths-crypto/src/lib.rs` — add gated `testing` module
- Create `crates/auths-crypto/src/testing.rs` — moved helpers

### Do NOT yet

- Do NOT update consumers (that's fn-12.8)
- Do NOT delete from `auths-test-utils` yet (that's fn-12.9)
- Keep `auths-test-utils/src/crypto.rs` intact for now
