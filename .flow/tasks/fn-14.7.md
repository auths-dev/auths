# Task: Round-Trip Export/Import and Validation

Build the public API for bidirectional translation and validate correctness with round-trip tests.

## Event type: `auths_verifier::keri::KeriEvent`

The translation layer uses `KeriEvent` from `auths-verifier` (Layer 1). This is the canonical event type with `s: String` (decimal sequence numbers). The `auths-id::Event` type is NOT used (would pull in Layer 3 deps).

Reference: `auths-verifier/src/keri.rs:308-318` -- `KeriEvent` enum definition.

## What to do

1. Implement `export_kel_as_cesr(codec, events: &[KeriEvent]) -> Result<CesrStream>`:
   - For each event: `serde_json::to_value()` -> `serialize_for_cesr()` -> collect
   - Assemble via `assemble_cesr_stream()`

2. Implement `import_cesr_to_events(codec, cesr_bytes: &[u8]) -> Result<Vec<KeriEvent>>`:
   - Parse self-framing JSON bodies by reading version strings
   - Extract hex size from `KERI10JSON{6 hex}_` to determine body length
   - Extract attached signature from CESR counter codes
   - Re-embed signature in `x` field as base64url (NOT hex)
   - Deserialize JSON back to `KeriEvent`

3. Internal helpers:
   - `extract_body_length(bytes)` -- find `KERI10JSON` marker, read 6-char hex size
   - `extract_attached_signature(codec, attachment_bytes)` -- parse `-AAB` counter + 88-char sig
   - `find_next_event_start(bytes, from)` -- find next `{` after attachments

4. Write integration tests in `tests/cases/roundtrip.rs`:
   - Build test KEL manually (generate keypairs, create ICP/ROT/IXN, sign them)
   - Export to CESR, verify stream structure
   - Import back, verify event count and field equality
   - Round-trip preserves key state (if `validate_kel` is accessible)
