## Description
## Create cargo xtask generate-schemas and validate-schemas commands

Add two new xtask subcommands for JSON Schema management.

### `cargo xtask generate-schemas`

Generates JSON Schema files from Rust types and writes them to `schemas/`:

```
schemas/
  attestation-v1.json
  identity-bundle-v1.json
  keri-icp-v1.json
```

Implementation:
1. Enable `schema` feature on auths-id and auths-keri in xtask's dev-dependencies
2. Use `schemars::schema_for::<Type>()` to generate schemas
3. Pretty-print JSON with stable key ordering (use `serde_json::to_string_pretty`)
4. Write to `schemas/` directory

### `cargo xtask validate-schemas`

Validates existing test fixtures against the committed schemas:
1. Read each schema from `schemas/`
2. Find test fixture JSON files (e.g., in `crates/auths-verifier/tests/fixtures/`)
3. Validate fixtures against schemas using `jsonschema` crate
4. Report pass/fail for each fixture

Add `jsonschema` as a dev-dependency of xtask.

### Files to modify
- `crates/xtask/Cargo.toml` (add auths-id, auths-keri with schema feature; add jsonschema)
- `crates/xtask/src/main.rs` (add subcommands)
- Create: `crates/xtask/src/schemas.rs`

### Verification
```bash
cargo xtask generate-schemas
ls schemas/
cargo xtask validate-schemas
```
