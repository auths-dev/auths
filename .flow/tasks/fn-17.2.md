## Description
## Load trust registry from JSON config in auths-oidc-bridge

Add trust registry loading to `auths-oidc-bridge`. The bridge reads a JSON file containing trust registry entries at startup.

### Feature gating

Gate behind a new `oidc-trust` feature flag (separate from `oidc-policy`):
```toml
oidc-trust = ["dep:auths-policy"]
```

Trust registry and policy evaluation are independent concerns. A bridge operator might want trust registry checks (coarse provider/repo/capability gating) without full policy expressions, or vice versa. Both depend on `auths-policy` for types, but enabling one doesn't require enabling the other.

Note: `oidc-policy` and `oidc-trust` both pull in `auths-policy` as a dependency â€” Cargo deduplicates this. No binary size overhead from having two features pointing to the same dep.

### Config changes

Add to `BridgeConfig` (feature-gated behind `oidc-trust`):

```rust
#[cfg(feature = "oidc-trust")]
pub trust_registry_path: Option<PathBuf>,
```

Add builder method `with_trust_registry_path()`.

### Loading

At bridge startup (`BridgeState::new()` or equivalent):
1. If `trust_registry_path` is `Some`, read the JSON file
2. Deserialize into `TrustRegistry` (validation happens on deser via `try_from`)
3. Store in `BridgeState` as `Option<TrustRegistry>`
4. Log at `info` level: "Loaded N trust registry entries for M providers"
5. Bridge does NOT start if path configured but file is invalid (fail-fast)

### JSON format

```json
{
  "entries": [
    {
      "provider_issuer": "https://token.actions.githubusercontent.com",
      "allowed_repos": ["myorg/myrepo-*"],
      "allowed_capabilities": ["sign:commit", "deploy:staging"],
      "max_token_ttl_seconds": 3600,
      "require_witness_quorum": null
    }
  ]
}
```

### Error handling

- File not found: return `BridgeError::Internal` with clear message including the path
- Invalid JSON / validation failure: return `BridgeError::Internal` with parse error details
- These errors prevent bridge startup â€” they're configuration errors, not runtime errors
