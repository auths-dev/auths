## Commit initial JSON Schema artifacts and add CI validation

Run `cargo xtask generate-schemas` and commit the output. Add a CI step that regenerates schemas and diffs against committed files.

### Commit schemas

Run `cargo xtask generate-schemas` and commit the generated files:
```
schemas/attestation-v1.json
schemas/identity-bundle-v1.json
schemas/keri-icp-v1.json
```

### CI step

Add to `.github/workflows/ci.yml`:

```yaml
- name: Schema validation
  run: |
    cargo xtask generate-schemas
    git diff --exit-code schemas/
    cargo xtask validate-schemas
```

This ensures:
1. Schemas are always up-to-date with the Rust types
2. Test fixtures conform to the schemas
