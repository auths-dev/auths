# fn-14.6 CESR stream assembly

## Description
# Task: CESR Stream Assembly

Assemble a complete CESR stream from serialized events and their detached signature attachments. This is the wire format a KERI-compliant parser can consume.

## Wire format

A single KERI event message:
```
{JSON event body}                           <- self-framing via version string
-AAB                                        <- counter code: 1 controller indexed signature
AA{86 chars base64url}                      <- the actual indexed signature (index 0)
```

A complete KEL stream is these messages concatenated. The version string's hex byte count enables framing.

## Counter codes

| Code | Meaning |
|------|---------|
| `-A##` | Count of indexed controller signatures |
| `##` is 2-char base64url count | e.g., `AB` = 1, `AC` = 2 |

So `-AAB` = "1 controller indexed signature follows".

## What to do

1. Define `AttachmentGroup` struct with `controller_signatures: Vec<String>`

2. Define `CesrStream` struct with `bytes: Vec<u8>` and `event_count: usize`

3. Implement `assemble_cesr_stream(codec, events: &[SerializedEvent]) -> Result<CesrStream>`:
   - For each event: write JSON body bytes, then if signature present:
     - Encode signature via `codec.encode_indexed_signature()`
     - Write counter code (`-AAB` for 1 sig)
     - Write the CESR-encoded signature

4. Implement `cesr_count_char(count: u8) -> char` helper for base64url count encoding

5. Write tests:
   - Single inception event: starts with `{`, followed by `-AAB` and 88-char signature
   - 3-event stream: 3 JSON bodies each followed by attachment
   - Total byte count = sum of body sizes + attachment sizes
   - Event with no signature omits counter code

## Acceptance
- Stream for a single inception event starts with `{` (JSON body) followed by `-AAB` and 88-char signature
- Stream for 3 events contains 3 JSON bodies, each followed by its attachment
- Total stream byte count matches sum of individual body sizes + attachment sizes
- A `CesrStream` with 0 signatures per event omits the counter code and signature
- All tests pass under `cargo nextest run -p auths-keri -E 'test(stream)'`
