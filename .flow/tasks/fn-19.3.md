## Description
## Add batch benchmarks (1000 events batch vs sequential)

Benchmark comparing batch vs sequential event appending.

### Benchmark location

Extend `crates/auths-storage/benches/registry.rs` with new benchmark functions.

### Benchmarks

1. **Sequential baseline** — append 1000 events one at a time via `append_event`. This is the existing behavior.
2. **Batch** — append 1000 events via single `batch_append_events` call.
3. **Mixed-prefix batch** — 1000 events across 100 different identities (10 events each).

Use `criterion` (already used in existing benchmarks) with `iter_with_setup` to create fresh registries per iteration.

### Event generation

Create helper to generate N valid sequential events for a given prefix:
- ICP event at seq 0
- IXN events at seq 1..N-1
- All properly signed with test keys

### Expected result

Batch should be significantly faster due to:
- Single Git commit instead of N commits
- Single lock acquisition instead of N
- In-memory tree staging vs N tree reads

### Files to modify
- `crates/auths-storage/benches/registry.rs`

### Verification
```bash
cargo bench -p auths-storage -- batch
# Should show batch is faster than sequential
```
