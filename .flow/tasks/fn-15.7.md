## Fix auths-core, auths-policy, and remaining crate production unwraps

Covers the crates not handled by fn-15.3 (auths-id), fn-15.4 (auths-verifier/storage), or fn-15.6 (auths-cli).

### Audit results (pre-triaged)

**auths-core — 21 production calls:**
- 8 already have `#[allow]` annotations with justification (crypto/provider_bridge.rs, crypto/secp256k1.rs x2, config.rs x2, storage/encrypted_file.rs x2, api/runtime.rs)
- 13 are `Mutex::lock().unwrap()` in `storage/memory.rs` — need `#[allow]` with `// INVARIANT: Mutex poisoning is fatal by design`
- 1 is `api/ffi.rs:201` — `.expect("FFI string inputs must be valid UTF-8")` — needs `#[allow]` with `// INVARIANT: FFI caller contract requires valid UTF-8`
- NOTE: The `panic!("Expected Ed25519 key")` at `agent/client.rs:662` is inside a `#[test]` function — not a production concern

**auths-policy — 1 production call (clean):**
- `compile.rs:134` — `serde_json::to_vec(expr).unwrap_or_default()` — safe, graceful fallback. No action needed.
- All ~128 other occurrences are in `#[cfg(test)]` blocks

**auths-infra-http — 3 production calls (infallible):**
- `witness_client.rs:32` — `.expect("failed to build reqwest client")` — reqwest builder with hardcoded Duration cannot fail
- `async_witness_client.rs:67,88` — same pattern
- Fix: Add `#[allow]` with `// INVARIANT: reqwest Client::builder() cannot fail with valid Duration params`

**auths-oidc-bridge — 4 production calls (mixed):**
- `rate_limit.rs:20-21` — `NonZeroU32::new(30).unwrap()` and `NonZeroU32::new(5).unwrap()` inside `unwrap_or()` — infallible (30 and 5 are non-zero). Fix: extract to `const` values.
- `config.rs:72` — `"0.0.0.0:3300".parse().unwrap()` in `Default` impl — infallible but fragile. Fix: extract to `const` or use `SocketAddr::from(([0,0,0,0], 3300))`.
- `error.rs:147` — HeaderValue conversion — infallible. Fix: `#[allow]` with comment.

**auths-telemetry — 1 production call (infallible):**
- `metrics.rs:19` — `.expect("failed to install Prometheus recorder")` — one-time startup. Fix: `#[allow]` with `// INVARIANT: called once at process startup`

**Confirmed clean (0 production unwraps):**
- auths-crypto
- auths-keri
- auths-infra-git
- auths-index

### Steps

1. **auths-core** (main work):
   - Add `#[allow(clippy::unwrap_used)]` with `// INVARIANT: Mutex poisoning is fatal by design` to all 13 `memory.rs` Mutex locks (can use a module-level allow)
   - Add `#[allow(clippy::expect_used)]` to `api/ffi.rs:201` with `// INVARIANT: FFI caller contract requires valid UTF-8`
   - Verify the 8 existing annotations still have justification comments

2. **auths-oidc-bridge**:
   - Extract `NonZeroU32::new(30).unwrap()` and `NonZeroU32::new(5).unwrap()` to module-level constants
   - Replace `"0.0.0.0:3300".parse().unwrap()` with `SocketAddr::from(([0,0,0,0], 3300))`
   - Add `#[allow]` to `error.rs:147` with INVARIANT comment

3. **auths-infra-http**: Add `#[allow]` annotations to reqwest builder expects (3 locations)

4. **auths-telemetry**: Add `#[allow]` to Prometheus init expect

5. **auths-policy**: Confirm clean — no changes needed (document in Done summary)
